# C# Method Overloading

Method overloading allows multiple methods to have the same name but different parameters.

The parameters must differ by:

- Number of parameters
- Type of parameters
- Order of parameters

## Syntax Example

```csharp
int MyMethod(int x)
float MyMethod(float x)
double MyMethod(double x, double y)
```

---

# Why Use Method Overloading?

Method overloading improves code readability and reuse.

Instead of creating different method names for similar tasks, you can use one method name for multiple versions.

Example:

Instead of:

```csharp
PlusMethodInt()
PlusMethodDouble()
```

You can simply use:

```csharp
PlusMethod()
```

---

# Example Without Overloading

```csharp
using System;

class Program
{
    static int PlusMethodInt(int x, int y)
    {
        return x + y;
    }

    static double PlusMethodDouble(double x, double y)
    {
        return x + y;
    }

    static void Main(string[] args)
    {
        int myNum1 = PlusMethodInt(8, 5);
        double myNum2 = PlusMethodDouble(4.3, 6.26);

        Console.WriteLine("Int: " + myNum1);
        Console.WriteLine("Double: " + myNum2);
    }
}
```

## Output

```text
Int: 13
Double: 10.56
```

# Example With Method Overloading

```csharp
using System;

class Program
{
    static int PlusMethod(int x, int y)
    {
        return x + y;
    }

    static double PlusMethod(double x, double y)
    {
        return x + y;
    }

    static void Main(string[] args)
    {
        int myNum1 = PlusMethod(8, 5);
        double myNum2 = PlusMethod(4.3, 6.26);

        Console.WriteLine("Int: " + myNum1);
        Console.WriteLine("Double: " + myNum2);
    }
}
```

## Output

```text
Int: 13
Double: 10.56
```

---

# How C# Chooses the Correct Method

C# automatically selects the correct overloaded method based on the arguments passed.

```csharp
PlusMethod(5, 10);       // Calls int version
PlusMethod(2.5, 3.8);    // Calls double version
```

# Overloading with Different Number of Parameters

```csharp
using System;

class Program
{
    static int Multiply(int x, int y)
    {
        return x * y;
    }

    static int Multiply(int x, int y, int z)
    {
        return x * y * z;
    }

    static void Main(string[] args)
    {
        Console.WriteLine(Multiply(2, 3));
        Console.WriteLine(Multiply(2, 3, 4));
    }
}
```

## Output

```text
6
24
```

# Overloading with Different Data Types

```csharp
using System;

class Program
{
    static int Display(int number)
    {
        return number;
    }

    static string Display(string text)
    {
        return text;
    }

    static void Main(string[] args)
    {
        Console.WriteLine(Display(100));
        Console.WriteLine(Display("Hello"));
    }
}
```

---

# Rules of Method Overloading

✅ Method names must be the same

✅ Parameters must be different

✅ Return type can be different, but it alone cannot overload a method

❌ Changing only the return type is NOT valid

Invalid example:

```csharp
int Test(int x)
double Test(int x)
```

# Real-World Example

```csharp
using System;

class Calculator
{
    static int Add(int a, int b)
    {
        return a + b;
    }

    static double Add(double a, double b)
    {
        return a + b;
    }

    static int Add(int a, int b, int c)
    {
        return a + b + c;
    }

    static void Main(string[] args)
    {
        Console.WriteLine(Add(2, 3));
        Console.WriteLine(Add(2.5, 3.7));
        Console.WriteLine(Add(1, 2, 3));
    }
}
```

# Advantages of Method Overloading

- Improves code readability
- Reduces duplicate method names
- Makes programs easier to maintain
- Supports polymorphism
