# C# Comments

Comments are used to explain C# code and make it easier to read.  
They can also be used to stop code from running during testing.

---

# Single-line Comments

Single-line comments start with `//`.

Anything after `//` on the same line is ignored by C#.

## Example 1: Comment Before Code

```csharp
// This is a comment
Console.WriteLine("Hello World!");
```

## Example 2: Comment After Code

```csharp
Console.WriteLine("Hello World!");  // This is a comment
```

---

# Multi-line Comments

Multi-line comments start with `/*` and end with `*/`.

Everything between them is ignored by C#.

## Example

```csharp
/* The code below will print the words Hello World
to the screen, and it is amazing */
Console.WriteLine("Hello World!");
```

---

# When to Use Each Type

| Comment Type | Syntax | Best Use |
|---|---|---|
| Single-line | `//` | Short comments |
| Multi-line | `/* */` | Longer explanations |

---

# Tips

- Use comments to explain complex logic.
- Keep comments short and clear.
- Avoid writing unnecessary comments.
- Update comments when changing code.

## Good Example

```csharp
// Calculate total price with tax
double total = price + tax;
```

## Bad Example

```csharp
// Set x to 5
int x = 5;
```

The second example is unnecessary because the code is already clear.
