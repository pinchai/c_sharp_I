# C# Exceptions

When executing C# code, different errors can occur:
- Coding errors made by the programmer
- Errors caused by invalid user input
- Other unexpected problems

When an error happens, C# usually stops the program and displays an error message. This is called an **exception**.

---

# try and catch

The `try` block lets you test code for errors.

The `catch` block handles the error if one occurs.

## Syntax

```csharp
try
{
    // Code to test
}
catch (Exception e)
{
    // Code to handle the error
}
```

---

# Example: Exception Without try-catch

```csharp
int[] myNumbers = {1, 2, 3};

Console.WriteLine(myNumbers[10]); // Error
```

This causes an exception because index `10` does not exist in the array.

Example error message:

```text
System.IndexOutOfRangeException:
Index was outside the bounds of the array.
```

---

# Example: Using try-catch

```csharp
try
{
    int[] myNumbers = {1, 2, 3};

    Console.WriteLine(myNumbers[10]);
}
catch (Exception e)
{
    Console.WriteLine(e.Message);
}
```

## Output

```text
Index was outside the bounds of the array.
```

---

# Custom Error Message

```csharp
try
{
    int[] myNumbers = {1, 2, 3};

    Console.WriteLine(myNumbers[10]);
}
catch (Exception e)
{
    Console.WriteLine("Something went wrong.");
}
```

## Output

```text
Something went wrong.
```

---

# finally Statement

The `finally` block always executes whether an error occurs or not.

```csharp
try
{
    int[] myNumbers = {1, 2, 3};

    Console.WriteLine(myNumbers[10]);
}
catch (Exception e)
{
    Console.WriteLine("Something went wrong.");
}
finally
{
    Console.WriteLine("The try-catch block is finished.");
}
```

## Output

```text
Something went wrong.
The try-catch block is finished.
```

# throw Keyword

The `throw` keyword is used to create custom exceptions.

## Example

```csharp
static void CheckAge(int age)
{
    if (age < 18)
    {
        throw new ArithmeticException(
            "Access denied - You must be at least 18 years old."
        );
    }
    else
    {
        Console.WriteLine("Access granted - You are old enough!");
    }
}

static void Main(string[] args)
{
    CheckAge(15);
}
```

## Output

```text
System.ArithmeticException:
Access denied - You must be at least 18 years old.
```

---

# Example With Valid Age

```csharp
CheckAge(20);
```

## Output

```text
Access granted - You are old enough!
```

# Common Exception Types in C#

| Exception Type | Description |
|---|---|
| `ArithmeticException` | Errors in arithmetic operations |
| `DivideByZeroException` | Division by zero |
| `IndexOutOfRangeException` | Invalid array index |
| `NullReferenceException` | Using a null object |
| `FileNotFoundException` | File cannot be found |
| `FormatException` | Invalid data format |

---

# Best Practices

- Use `try-catch` to prevent program crashes
- Use meaningful error messages
- Use `finally` for cleanup tasks
- Throw custom exceptions when validating data

# Exercise

## Exercise 1

Write a program that:
- Divides two numbers
- Uses `try-catch` to handle division by zero

---

## Exercise 2

Create a method called `CheckNumber()` that:
- Throws an exception if the number is negative
- Prints `"Valid Number"` otherwise

---

# Exercise Solution

## Solution 1

```csharp
try
{
    int x = 10;
    int y = 0;

    int result = x / y;

    Console.WriteLine(result);
}
catch (DivideByZeroException e)
{
    Console.WriteLine("Cannot divide by zero.");
}
```

---

## Solution 2

```csharp
static void CheckNumber(int number)
{
    if (number < 0)
    {
        throw new Exception("Negative numbers are not allowed.");
    }

    Console.WriteLine("Valid Number");
}

static void Main(string[] args)
{
    try
    {
        CheckNumber(-5);
    }
    catch (Exception e)
    {
        Console.WriteLine(e.Message);
    }
}
```
